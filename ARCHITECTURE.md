### Component Details

#### **UI Layer**
- `HomeScreen`: Main navigation, connection status
- `BluetoothScanScreen`: Device discovery, connection
- `RfidOperationsScreen`: Card operations, data display
- `WriteCardScreen`: Data input, write operations

#### **Service Layer**
- `BleService`: 
  - Manages BLE connection lifecycle
  - Handles device scanning
  - Sends/receives raw data
  - Connection state management

- `RfidService`:
  - Implements RFID protocol
  - Parses ESP32 responses
  - Provides high-level card operations
  - Manages card data model

#### **Model Layer**
- `RfidCard`: Card data structure
  - UID storage
  - Block data mapping
  - Hex/ASCII conversion

## ğŸ”Œ Communication Protocol

### BLE UART Protocol

**Transport**: Bluetooth Low Energy
**Profile**: Nordic UART Service (NUS)
**Format**: ASCII text with newline terminator

#### Message Format
```
Command:  <CMD>[:<PARAM1>:<PARAM2>:...]\n
Response: <TYPE>:<MESSAGE>\n
```

#### Examples
```
App â†’ ESP32: "READ\n"
ESP32 â†’ App: "UID:04A1B2C3\n"
ESP32 â†’ App: "DATA:4:48656C6C6F\n"
ESP32 â†’ App: "READ_COMPLETE\n"
ESP32 â†’ App: "OK:Card read successfully\n"
```

### RFID Protocol

**Interface**: SPI
**Frequency**: 13.56 MHz
**Standard**: ISO/IEC 14443 Type A

#### Authentication
```
1. Card detection (PICC_IsNewCardPresent)
2. Read serial (PICC_ReadCardSerial)
3. Authenticate (PCD_Authenticate)
   - Sector select
   - Key A or Key B
   - Default: FFFFFFFFFFFF
4. Read/Write operation
5. Halt card (PICC_HaltA)
6. Stop crypto (PCD_StopCrypto1)
```

```cpp

  sendResponse("NO_CARD");
  return;

  sendResponse("ERROR:Authentication failed");
  return;
}
```

### Recovery Strategies
1. **Connection lost**: Auto-reconnect
2. **Card removed**: Retry operation
3. **Auth failed**: Try default keys
4. **Write failed**: Verify and retry
5. **Timeout**: Cancel and notify
